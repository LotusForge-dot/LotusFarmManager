# 8. データ保存仕様

Version 5.0

---

## 8.1 概要

Lotus Farm Manager は、ブラウザの LocalStorage を利用してデータを保存する。

サーバーは使用せず、すべてのデータは端末内で管理される。

保存形式は JSON とする。

---

## 8.2 保存対象

現在保存するデータは以下のとおり。

| 保存キー | 内容 |
|----------|------|
| records | 作業記録 |
| fieldMaster | 田んぼマスタ |
| workMaster | 作業マスタ |
| materialMaster | 資材マスタ |
| fertilizerPlans | 施肥設計 |

---

## 8.3 保存処理

データ保存は storage.js の saveData() を経由して行う。

```
データ変更
    │
    ▼
saveData()
    │
    ▼
JSON.stringify()
    │
    ▼
LocalStorage
```

保存処理は、各画面から直接 LocalStorage を操作せず、saveData() を共通利用する。

---

## 8.4 読込処理

アプリ起動時、およびデータ復元時には loadData() を実行する。

```
アプリ起動
    │
    ▼
loadData()
    │
    ▼
LocalStorage
    │
    ▼
JSON.parse()
    │
    ▼
各配列へ展開
```

---

## 8.5 バックアップ

バックアップでは、保存対象の全データを JSON として出力する。

```
各データ取得
    │
    ▼
JSON生成
    │
    ▼
ファイル保存
```

---

## 8.6 データ復元

復元では、バックアップファイルを読み込み、LocalStorage を更新する。

```
JSON読込
    │
    ▼
内容確認
    │
    ▼
LocalStorage更新
    │
    ▼
loadData()
    │
    ▼
画面再表示
```

---

## 8.7 データ管理方針

データ管理は以下を基本方針とする。

- 保存処理は saveData() に統一する。
- 読込処理は loadData() に統一する。
- LocalStorage は storage.js のみが直接操作する。
- データは JSON 形式で管理する。
- データ変更後は必要に応じて画面を再描画する。

---

## 8.8 エラー対策

データの読込時は、保存データが存在しない場合や形式が異なる場合でも、アプリケーションが停止しないよう初期値を設定する。

将来的には、バックアップファイルのバージョンチェックやデータ整合性チェックの追加を検討する。
