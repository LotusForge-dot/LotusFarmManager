# 7. 処理フロー

Version 5.0

---

## 7.1 概要

Lotus Farm Manager は、各画面でユーザー操作を受け付け、必要に応じてデータを保存・読込しながら画面を更新する。

画面表示は app.js、データ保存は storage.js が担当する。

## 7.9 共通処理

画面更新は、データ変更後に対応する render○○() 関数を呼び出して再描画を行う。

保存処理は saveData() を経由して LocalStorage に保存する。

これにより、画面表示とデータ管理を分離している。

---

## 7.11 設計方針

Lotus Farm Manager では、

- データ変更と画面描画を分離する
- 保存処理は saveData() に統一する
- 画面更新は render○○() に統一する
- マスタデータを共通利用する

ことを基本方針とする。

これにより、処理の統一性と保守性を確保している。
