# 3. ファイル構成

Version 5.0

---

## 3.1 概要

Lotus Farm Manager は、役割ごとに JavaScript ファイルを分割して構成する。

各ファイルは責務を明確に分離し、保守性・可読性の向上を目的としている。

---

## 3.2 ファイル一覧

| ファイル | 役割 |
|----------|------|
| index.html | アプリケーションのエントリーポイント |
| style.css | 画面デザイン・レイアウト |
| app.js | UI・画面表示・イベント処理 |
| master.js | 各種マスタ管理 |
| storage.js | LocalStorageへの保存・読込 |

---

## 3.3 app.js

### 役割

ユーザーインターフェース全体を管理する。

主な担当は以下のとおり。

- メニュー画面
- 作業記録
- 作業履歴
- 施肥設計
- 画面切替
- 入力チェック
- 一覧表示
- 集計表示

app.js は「ユーザーが操作する画面」を担当する。

---

## 3.4 master.js

### 役割

各種マスタデータを管理する。

管理対象は以下のとおり。

- 田んぼマスタ
- 作業マスタ
- 資材マスタ

登録・編集・削除・一覧表示を担当する。

---

## 3.5 storage.js

### 役割

LocalStorage を利用したデータ保存を担当する。

管理対象は以下のとおり。

- 作業記録
- 田んぼマスタ
- 作業マスタ
- 資材マスタ
- 施肥設計
- バックアップ
- 復元

データは JSON 形式で保存する。

---

## 3.6 今後の方針

機能追加によりソースコードが肥大化した場合は、新たな JavaScript ファイルへ責務を分割する。

各ファイルは一つの責務を持つことを原則とする。
