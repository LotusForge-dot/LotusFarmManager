# FUNCTIONS

Version 1.0

---

# 0. はじめに

## 0.1 本書について

本書は、Lotus Farm Manager の関数仕様書である。

DESIGN がシステム全体の設計を説明するドキュメントであるのに対し、本書では各 JavaScript ファイル内の関数について説明する。

本書を参照することで、各関数の役割や処理内容を把握し、ソースコードを効率よく理解できることを目的とする。

---

## 0.2 対象ファイル

現在の対象ファイルは以下のとおりである。

| ファイル | 内容 |
|----------|------|
| app.js | 画面表示・イベント処理 |
| master.js | 各種マスタ管理 |
| storage.js | データ保存・読込 |

今後 JavaScript ファイルが追加された場合は、本書も同様に追加する。

---

## 0.3 記載内容

各関数について、以下の内容を記載する。

- 関数名
- 目的
- 引数
- 戻り値
- 呼び出し元
- 呼び出し先
- 処理概要
- 使用画面
- 備考（必要に応じて）

---

## 0.4 記載ルール

本書では、ソースコードそのものは掲載しない。

コードの内容ではなく、「その関数が何を担当するのか」を記載する。

関数は、ソースコードと同じ順番で記載する。

---

## 0.5 更新ルール

関数の追加・削除・名称変更・責務変更を行った場合は、本書も同時に更新する。

コードと関数仕様書の内容は常に一致させる。

---

## 0.6 DESIGNとの関係

Lotus Farm Manager のドキュメントは以下の役割で構成する。

```
DESIGN
    │
    ├─ システム全体
    │
    ▼
FUNCTIONS
    │
    ├─ 関数仕様
    │
    ▼
ソースコード
```

DESIGN を読むことでシステム全体を理解し、FUNCTIONS を読むことで各関数の役割を理解できる構成とする。

---

## 0.7 開発ルール

Lotus Farm Manager は、以下の手順で開発を行う。

```
コード修正
    │
    ▼
動作確認
    │
    ▼
CHANGELOG更新
    │
    ▼
DESIGN更新（必要に応じて）
    │
    ▼
FUNCTIONS更新（必要に応じて）
```

ドキュメントとソースコードは常に同期した状態を維持する。

---

## 0.8 本書の目的

本書は、現在の開発者だけでなく、将来の開発者や新しい開発環境でも短時間でシステムを理解できることを目的とする。

コードだけでは伝わりにくい設計意図や関数の役割を明文化し、保守性・可読性・継続的な開発効率の向上を目指す。
