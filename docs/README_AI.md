# Lotus Farm Manager

AI引き継ぎドキュメント

Version 1.0

---

# プロジェクト概要

Lotus Farm Manager はレンコン栽培管理アプリである。

目的は

・作業記録
・作業履歴
・施肥設計
・各種マスタ管理

を一元管理することである。

---

# 最初に読むファイル

① DESIGN

システム全体を理解する。

↓

② FUNCTIONS

↓

③ CHANGELOG

現在どこまで開発が進んでいるか確認する。

↓

④ ソースコード

必要な箇所のみ読む。

# ファイル構成

データ保存・読込

---

# 開発ルール

・設計変更 → DESIGN更新

・関数追加・変更 → FUNCTIONS更新

・実装変更 → CHANGELOG更新

---

# コーディングルール

・既存の命名規則を維持する。

・render○○() は画面表示のみ担当する。

・save○○() は保存のみ担当する。

・load○○() は読込のみ担当する。

・業務ロジックと保存処理を混在させない。

---

# AIへのお願い

コードを書く前に

・DESIGN
・FUNCTIONS
・CHANGELOG

を確認すること。

推測で実装を変更しない。

既存設計との整合性を優先する。

# AIへの最重要事項

Lotus Farm Managerでは

「動けばよい」

ではなく

「設計を維持する」

ことを優先する。

新しい処理を追加する際は、

既存設計・責務分離・命名規則を崩さないこと。

LFM設計哲学
① データ構造はなるべく変えない。
保存形式は資産。 新機能のたびに変えない。
② 新機能はデータ加工で実現する。
保存データを増やす前に、
filter
map
集計
表示
で解決できないか考える。
③ 入力は最小限。
人が入力する量を減らし、 計算できるものは全部計算する。
④ マスタを活用する。
面積、標準使用量、箱代、肥料成分...
一度登録した情報は何度も使う。
⑤ 表示は自由。保存はシンプル。
画面ごとに好きな形へ加工する。
