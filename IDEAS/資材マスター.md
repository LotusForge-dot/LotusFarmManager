# 作業マスタのカテゴリ化（案）

## 概要

現在は作業名のみで管理しているが、カテゴリを持たせることで入力画面・施肥設計・履歴検索を共通化できる。

## 現在

```javascript
{
    name: "追肥①"
}
```

## 変更案

```javascript
{
    category: "fertilizer",
    name: "追肥①"
}
```

## イメージ

肥料
- 元肥
- 追肥①
- 追肥②
- 追肥③

葉面散布
- 通常
- 台風後
- 高温対策

除草剤
- 初期
- 中期
- 後期

その他
- 写真
- メモ

## メリット

- タブごとにカテゴリで自動表示できる
- 入力画面を共通処理にできる
- 施肥設計から自動入力しやすい
- 新しい作業はマスタ追加だけで対応可能
- 履歴・検索・集計でもカテゴリを利用できる
- コードの分岐を減らせる

## 懸念点

- 作業マスタのデータ構造変更が必要
- 作業登録画面の修正が必要

※ 保存・読込は structuredClone() を使用しているため、データ項目が増えても影響は比較的小さいと思われる。

## 実装タイミング

入力画面を本格実装する前に検討。
